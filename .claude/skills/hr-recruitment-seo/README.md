# 人材紹介SEO記事作成スキル

人材紹介・転職業界に特化したSEO記事を、Claude Codeで作成するためのスキルです。

## できること

- **企業評判記事**の作成（「〇〇 評判」「〇〇 やばい」系）
- **転職エージェント比較記事**の作成（「〇〇業界 転職エージェント おすすめ」系）
- **職種・業界解説記事**の作成（「〇〇 年収」「〇〇 将来性」系）
- **転職ノウハウ記事**の作成（「面接対策」「職務経歴書 書き方」系）
- 口コミ・評判の引用風表現とデータによる裏付け
- 転職エージェント紹介カードの挿入
- FAQ（よくある質問）セクションの自動生成
- Nano Banana（Gemini API）による挿絵の自動生成

## ディレクトリ構成

```
.claude/skills/hr-recruitment-seo/
├── README.md                 ← このファイル（使い方ガイド）
├── SKILL.md                  ← スキル定義（Agentが読む指示書）
├── examples/                 ← サンプル記事
│   └── sample-01-company-review.md  ← 企業評判記事のサンプル
└── templates/
    └── article-template.md   ← 記事テンプレート（A/B/C 3タイプ）
```

| ファイル / フォルダ | 役割 |
|-------------------|------|
| `SKILL.md` | Agentへの指示。記事タイプ別の書き方ルール、品質基準、出力フォーマットを定義 |
| `examples/` | サンプル記事。Agentがトーン・構成・文体の参考にする |
| `templates/` | 記事の構成テンプレート。企業評判・エージェント比較・職種解説の3タイプ |

## セットアップ

### 1. スキルの配置

`.claude/skills/hr-recruitment-seo/` フォルダがプロジェクトルートに配置されていれば、特別な設定は不要です。

### 2. 自社記事の追加（推奨）

`examples/` フォルダに自社の既存記事を `.md` 形式で追加してください。Agentはこれらを参考にして、自社のトーンや文体に合った記事を生成します。

```bash
# 例：自社の記事を追加
cp 自社の企業評判記事.md .claude/skills/hr-recruitment-seo/examples/
```

追加する記事は、自社で「よく書けている」と思う記事を選んでください。記事タイプごとに1〜2本あると精度が上がります。

### 3. 画像生成の設定（任意・推奨）

挿絵の自動生成を有効にするには、Gemini APIキーを環境変数に設定してください。

```bash
export GEMINI_API_KEY="your-api-key-here"
```

APIキーは [Google AI Studio](https://aistudio.google.com/apikey) から無料で取得できます。

APIキーがなくても記事作成は可能です。その場合、画像の挿入箇所に生成プロンプト（HTMLコメント形式）が出力されます。

### 4. プロダクトコンテキストの設定（任意）

`.claude/product-marketing-context.md` を作成しておくと、自社のサービス情報（紹介するエージェント、ターゲット読者など）を毎回説明する手間が省けます。

```markdown
# product-marketing-context.md の例

## サービス概要
- サービス名：〇〇転職ナビ
- ターゲット：20代〜30代のIT業界転職希望者

## 紹介するエージェント
- レバテックキャリア（アフィリエイトリンク：https://...）
- ワークポート（アフィリエイトリンク：https://...）

## CTA
- メインCTA：自社転職サービスへの登録誘導
- サブCTA：LINE登録で転職ガイドブック無料配布
```

```

```


### 記事タイプ別の指示例

#### 企業評判記事（タイプA）

```
人材紹介SEO記事を作成してください。

- 記事タイプ：企業評判
- ターゲットキーワード：ハイマックス 評判
- 関連キーワード：ハイマックス やばい, ハイマックス 年収, ハイマックス 口コミ
- 想定読者：ハイマックスへの転職を検討中の20代〜30代エンジニア
- CTA：転職エージェントへの無料相談誘導
- 紹介エージェント：レバテックキャリア、ワークポート、マイナビIT AGENT
```

#### 転職エージェント比較記事（タイプB）

```
人材紹介SEO記事を作成してください。

- 記事タイプ：エージェント比較
- ターゲットキーワード：コンサル 転職エージェント おすすめ
- 想定読者：コンサル業界への転職を考えている20代〜30代のビジネスパーソン
- 紹介エージェント数：5社
- 目標文字数：6,000文字
```

#### 職種・業界解説記事（タイプC）

```
人材紹介SEO記事を作成してください。

- 記事タイプ：職種解説
- ターゲットキーワード：データサイエンティスト 年収
- 関連キーワード：データサイエンティスト 将来性, 未経験 データサイエンティスト
- 想定読者：データサイエンティストへのキャリアチェンジを考えている方
```

## カスタマイズ

### サンプル記事の差し替え

`examples/` 内のサンプル記事は自由に差し替え・追加できます。

- ファイル形式は `.md`（Markdown）
- ファイル名は自由
- 冒頭にYAMLフロントマター（title, description, target_keyword, article_type）を付けると参照精度が上がります

#### おすすめのサンプル追加方法

| 記事タイプ | 推奨サンプル数 | 例 |
|-----------|-------------|-----|
| 企業評判 | 2〜3本 | SES企業、大手SI、Web系それぞれ1本ずつ |
| エージェント比較 | 1〜2本 | 業界特化型と総合型それぞれ1本 |
| 職種解説 | 1〜2本 | IT職種と非IT職種それぞれ1本 |

### テンプレートの編集

`templates/article-template.md` の構成を自社の記事フォーマットに合わせて編集できます。

カスタマイズ例：
- 紹介するエージェント数を固定する
- 「年収テーブル」セクションを必須にする
- 独自のCTA定型文を埋め込む
- 「この記事の監修者」セクションを追加する

### SKILL.md の調整

記事の品質基準やルールを変更したい場合は、`SKILL.md` を直接編集してください。

変更例：
- 紹介エージェントのデフォルトリストを設定する
- 特定の表現ルールを追加する（自社のライティングガイドラインに合わせる）
- 口コミの引用スタイルを変更する
- CTAの定型文を設定する

## よくある質問

### Q: 既存のseo-articleスキルとの違いは？

`seo-article` は汎用SEO記事用のスキルです。`hr-recruitment-seo` は人材紹介業界に特化しており、以下の機能が追加されています：
- 口コミ・評判の引用風表現テンプレート
- 転職エージェント紹介カードの定型フォーマット
- 企業データ（年収、残業時間、離職率）の表現ルール
- 向いている人・向いていない人の比較表
- 人材紹介業界の法的・倫理的注意事項

### Q: 口コミは実在するものを使っていますか？

サンプル記事の口コミは、口コミサイトの傾向を踏まえた引用風の表現です。実際の口コミをそのまま転載することは著作権の観点から避けています。記事公開前に、口コミの内容が事実に基づいているか確認してください。

### Q: 紹介するエージェントはどう決めればいいですか？

以下の観点で選定することを推奨します：
- 対象読者（業界・職種・年代）にマッチしているか
- アフィリエイトプログラムがある場合はその対象か
- 自社サービスの場合は自然に紹介できるか
- 読者にとって本当に役立つサービスか

### Q: 数値データ（年収、残業時間など）はどこから取得すればいいですか？

以下の情報源を推奨しています：
- **有価証券報告書**：上場企業の平均年収・平均勤続年数
- **OpenWork / Vorkers**：口コミベースの年収・残業時間
- **doda / マイナビ**：業界別平均年収データ
- **厚生労働省**：業界別の統計データ
- **企業の採用ページ・IR資料**：公式の情報

記事内では必ず出典を明記してください。

### Q: 生成された記事はそのまま公開できますか？

公開前に以下を必ず確認してください：
- 数値データ（年収、残業時間、求人数など）の最新性と正確性
- 口コミ内容が事実に基づいているか
- 紹介エージェントのリンクURLが正しいか
- 特定企業を不当に貶める表現がないか
- 景品表示法に違反する表現（根拠のない「No.1」等）がないか
- 生成画像の品質（不自然な箇所がないか）

### Q: 画像が生成されない場合は？

`GEMINI_API_KEY` が未設定の場合、記事内の挿入箇所に `<!-- IMAGE: ... -->` 形式でプロンプトが出力されます。このプロンプトを使って手動生成できます。
